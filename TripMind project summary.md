# 🌍 TripMind - AI 기반 여행 계획 플랫폼

## 📊 프로젝트 개요

**TripMind**는 여러 API와 AI를 통합하여 항공편 예약, 호텔 예약, 지능형 일정 생성, 환율 변환을 제공하는 종합 여행 계획 애플리케이션입니다.

---

## 🛠️ 기술 스택

### **Frontend**
- **React** (컴포넌트 기반 UI)
- **React Router** (SPA 라우팅)
- **Tailwind CSS** (스타일링)
- **Lucide React** (아이콘)
- **Recharts** (차트 라이브러리)

### **Backend**
- **Flask** (메인 서버)
- **FastAPI** (MCP 서버)
- **MySQL** (데이터베이스)
- **JWT** (인증)

### **AI & External APIs**
- **Google Gemini 2.5 Flash** (LLM)
- **AGODA API** (항공편/호텔)
- **Google Places API** (POI 검색)
- **WeatherAPI** (날씨 정보)
- **KEB 한국외환은행 API** (환율)

### **Architecture**
- **MCP (Model Context Protocol)** 서버
- **RESTful API** 설계
- **JWT 기반 인증 시스템**

## ✅ **완료된 기능**

### **🎯 핵심 기능**
1. **여행 계획 생성**
   - 자연어 입력 파싱 (목적지, 날짜, 인원, 관심사)
   - AI 기반 여행 스타일 추론 (휴양형, 관광형, 미식형, 쇼핑형, 액티비티형)
   - 통합 API 호출 (항공편, 호텔, POI, 날씨)

2. **실시간 비용 계산 시스템**
   - 항공편 + 호텔 + 기타 여행비 (식비, 교통비, 입장료 등)
   - 여행 스타일별 차등 비용 계산 (휴양형: 7만원/일, 미식형: 12만원/일 등)
   - 실시간 업데이트 및 데이터베이스 저장

3. **동적 일정 편집**
   - **드래그 앤 드롭** 일정 재배치
   - POI 선택 및 추가
   - 항공편 시간 기반 자동 스케줄 조정

4. **travel_style별 동적 데이터 시각화**
   - 도넛 차트 (활동 비율)
   - 휴양형: 휴식 60%, 관광 25%, 쇼핑 15%
   - 관광형: 관광 70%, 휴식 20%, 쇼핑 10%
   - 미식형: 맛집 50%, 관광 30%, 휴식 20%

### **🎨 사용자 인터페이스**
1. **반응형 디자인** (모바일/데스크톱 대응)
2. **다크 모드 지원** (CSS 변수 기반)
3. **인터랙티브 컴포넌트**
   - 탭 네비게이션
   - 모달 다이얼로그
   - 로딩 스피너
   - 호버 효과

### **🔐 인증 & 데이터 관리**
1. **JWT 토큰 인증** (24시간 유효)
2. **여행 계획 저장/조회** 시스템
3. **편집 모드** (일정 수정, 제목 편집)
4. **실시간 데이터 동기화**

### **🌐 API 통합**
1. **AGODA API** - 항공편/호텔 검색
2. **Google Places** - POI 검색 (평점 4.0+ 필터링)
3. **날씨 API** - 날짜별 날씨 정보
4. **환율 API** - 실시간 환율 변환

## ⚠️ **기술적 한계점**

### **🏗️ 아키텍처**
1. **단일 서버 구조**
   - 확장성 제한 (로드 밸런싱 필요)
   - 장애 시 전체 서비스 중단

2. **데이터베이스 설계**
   - **정규화 부족** (중복 데이터 존재)
   - **인덱싱 최적화** 필요
   - **백업/복구 전략** 미흡

### **🔌 API 통합**
1. **에러 처리**
   - **외부 API 장애 시 fallback** 로직 부족
   - **타임아웃 처리** 개선 필요

2. **데이터 일관성**
   - **실시간 동기화** 이슈 (항공편 가격 변동 등)
   - **캐시 무효화** 전략 필요

### **📱 프론트엔드**
1. **상태 관리**
   - **Redux/Zustand** 도입 필요 (복잡성 증가 시)
   - **window 객체 의존** (임시 데이터 저장)

2. **성능**
   - **코드 스플리팅** 미적용
   - **이미지 lazy loading** 부족
   - **번들 크기 최적화** 필요

### **🔒 보안**
1. **민감 정보 노출**
   - **API 키 클라이언트 노출** (환경변수 필요)
   - **HTTPS 강제화** 필요

2. **입력 검증**
   - **XSS/SQL Injection** 방어 강화 필요
   - **파라미터 validation** 개선
